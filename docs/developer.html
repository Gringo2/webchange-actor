Developer Guide & Extensibility

A guide for engineers looking to customize or extend the SWIM codebase.

Figure 8: Core vs. Extension Points

Project Structure

src/
├── core/           # The Engine (LCS Diff, Fetcher, Preprocessor)
├── intelligence/   # Logic (Classifier, Scorer, Deduplicator)
├── notifications/  # Output channels (Slack, etc.)
├── storage/        # Persistence (HistoryStore)
└── config.ts       # Preset definitions

How to Add a New Preset

Presets are defined in src/config.ts. To add a new one:

  1. Open src/config.ts.
  2. Add a new entry to the PRESETS record.
  3. Update src/types.ts to include your new preset name in the Zod enum.

Adding a New Output Channel

SWIM's modular design makes it easy to add new alert destinations (e.g., Discord, Email, Microsoft Teams).

  1. Create a new file in src/notifications/ (e.g., discord.ts).
  2. Implement an async notify method.
  3. Import and call it inside the targetUrls loop in src/main.ts.

Local Development

To run the actor locally for testing:

npm install
npm run start:dev

This uses tsx to run the TypeScript code directly without a full build step.
