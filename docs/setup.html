Getting Started & Configuration

A step-by-step guide to deploying your first intelligent web monitor.

Figure 7: The Monitor Setup Process

Quick Start (3 Minutes)

  1. Create an Apify Task based on the SWIM Actor.
  2. Set the targetUrl to the product or page you want to track.
  3. Select a preset (e.g., competitor-pricing).
  4. Click Run to establish your first baseline.

Input Reference Guide

Parameter Type Default Description
targetUrl URL or Array Required The URL(s) of the page(s) you wish to monitor. Supports batch processing.
preset Enum competitor-pricing One of: competitor-pricing, inventory-tracker, seo-intelligence, generic.
useVisualProof Boolean false Enable context-aware screenshots on detected changes.
slackWebhookUrl URL (String) null Your Slack Incoming Webhook URL for rich alerts.
useAi Boolean false Summarize changes using LLM (requires API Key).
cooldownPeriodMinutes Number 60 Prevents duplicate alerts for the same change within this window.

Dataset Output Format

When a change is detected, SWIM pushes a detailed analysis result to the Apify Dataset. Below is an example of the structured JSON output:

{
  "url": "https://example.com/product",
  "changeDetected": true,
  "severityScore": 85,
  "classification": "price_drop",
  "summary": "AI Summary: Price dropped from $99 to $79.",
  "visualProofUrl": "https://api.apify.com/v2/key-value-stores/...",
  "diff": [
    { "type": "removed", "content": "$99.00", "selector": ".price" },
    { "type": "added", "content": "$79.00", "selector": ".price" }
  ],
  "#debug": {
    "runId": "abc123xyz",
    "preset": "competitor-pricing",
    "engine": "lcs-v1.1"
  }
}

This structured format allows for easy integration into custom dashboards, ERPs, or external databases.
