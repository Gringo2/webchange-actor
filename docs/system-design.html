Persistence Strategy

Named Key-Value Stores

To support scheduled runs on Apify, SWIM utilizes named stores:

  • SWIM-SNAPSHOTS: Stores the baseline HTML for comparison.
  • SWIM-HISTORY: Maintains a version history of last N changes.
  • SWIM-DEDUPE: Stores event hashes to prevent duplicate alerts within a cooldown period.

Semantic Classification

Heuristic Scoring

The system calculates a Severity Score (0-100) by combining:

  • Selector Priority (e.g., .price > .footer)
  • Keyword Matches (e.g., "Sold Out", "Sale")
  • Numeric Change Detection (e.g., price drops)
