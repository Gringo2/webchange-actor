System Architecture

A deep dive into the high-level components and connectivity of the Semantic Web Intelligent Monitor.

Figure 1: Component Interaction Map

The Core Pipeline

1. Input & Batch Sequencing

SWIM V1.2 introduces a High-Scale Batch Loop. The actor validates the list of target URLs and processes each in sequence, maintaining isolated state stores for every target while sharing global proxy resources.

2. Signal Processing (Normalization)

The Preprocessor strips noise. It targets dynamic attributes (IDs, test tags) and collapses whitespace. This is critical for the LCS engine to function without false positives.

3. Comparison (LCS Engine)

Unlike standard scrapers that compare indexed arrays, SWIM uses the Longest Common Subsequence (LCS) algorithm. This allows the system to detect shifting content (like new products being prepended to a list) as a single addition rather than a massive layout shift.

4. Intelligence & Decisions

The Scorer and Classifier evaluate the diffs. They decide if a change is "Sensitive" (Pricing/Inventory) or "Structural".
