Security & Trust

How SWIM handles sensitive data, API keys, and secure web access.

Data Privacy & Handling

BYOK (Bring Your Own Key) Model

SWIM does not charge for AI usage. Instead, it uses a BYOK model for OpenAI and Anthropic integrations.

  • Transparency: You know exactly what you are paying for AI.
  • Control: Your API keys are never stored by SWIM; they are passed directly to the provider.
  • Isolation: Each user uses their own limits and billing.

Secrets Management

All sensitive inputs (Slack Webhooks, AI Keys) should be managed via Apify Secret Inputs.

  • These values are encrypted at rest by Apify.
  • They are only decrypted inside the Actor's sandbox during runtime.
  • They are never logged to the console by the SWIM core.

Network Security

Encrypted Proxy Traffic

When using proxyConfiguration, all traffic between SWIM and the Apify Proxy is encrypted. This prevents local network snooping on your target URLs and data.

Sandboxed Execution

SWIM runs inside a multi-stage Docker container (Node 22) on Apify's secure infrastructure. Each run is ephemeral and isolated from other users.
