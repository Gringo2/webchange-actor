Operations & Production Strategy

How to scale SWIM for multiple competitors and persistent scheduled runs.

Figure 6: Multi-Task & Scheduling Architecture

Monitoring Multiple Competitors

The Direct Batch Model (Recommended)

SWIM now supports monitoring multiple competitors in a single run. Simply provide a list of URLs in the targetUrl field.

  • Efficiency: Lower resource overhead by processing all targets in one session.
  • Consolidated Alerts: Get summaries for all competitors in a single batch flow.
  • Severity Filtering: Use minSeverityToAlert to ignore minor shifts across the entire fleet.

Individual Task Isolation

For high-priority competitors requiring unique settings (different Slack channels or AI models), use individual Apify Tasks.

Scheduling & Persistence

Scheduled Checkpoints (Cron)

Using Apify's built-in Scheduler, you can trigger SWIM checks every hour, day, or minute.

  • Snapshots: SWIM automatically retrieves the "last successful" HTML from SWIM-SNAPSHOTS.
  • Idempotency: If a scheduled run fails due to network issues, the system retains the previous "Success" state safely.
  • Stateful History: Each competitor's history is maintained in a dedicated slot, ensuring no data cross-contamination between different monitor tasks.
